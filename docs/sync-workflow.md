# 每轮自动同步流程说明

## 概述

Git Auto Sync 以固定间隔（默认 2 分钟）循环执行，每轮对 `repos.txt` 中所有未注释的仓库逐一执行同步操作。

---

## 单轮同步完整流程

### 1. 读取配置

每轮开始时重新读取 `config/sync-settings.txt`，获取两个参数：

| 参数 | 含义 | 默认值 |
|---|---|---|
| `INTERVAL` | 同步间隔（分钟） | 10 |
| `KEEP_RECENT` | 最近日志保留的轮数 | 5 |

### 2. 防重复启动

检测是否已有同步进程在运行，若存在则跳过本轮退出。

- **Windows**：通过 WMI 查询正在运行的同名 `cmd.exe` 进程数
- **Linux/macOS**：通过 PID 锁文件 (`git-auto-sync.lock`) 判断

### 3. 检查并更新分支配置（每轮执行）

**每轮同步都会执行此步骤**，而非仅首次。分为两种情况：

#### 情况 A：`branches.txt` 不存在

条件：`branches.txt` 不存在 **且** `repos.txt` 中有未注释的有效仓库。

1. 遍历每个仓库，执行 `git branch` 获取所有本地分支
2. 以当前检出分支作为默认生效分支，其余分支注释列出
3. 生成 `config/branches.txt` 并打开编辑器供用户修改
4. 退出本轮，等下一轮生效

此情况仅在首次运行时触发一次。

#### 情况 B：`branches.txt` 已存在

每轮都会遍历 `repos.txt` 中的所有仓库，逐一检查是否已在 `branches.txt` 中有条目：
- **已有条目** → 无操作
- **新仓库**（在 `repos.txt` 中但不在 `branches.txt` 中）→ 自动追加该仓库的分支配置

这意味着：用户新增仓库到 `repos.txt` 后，无需手动编辑 `branches.txt`，下一轮同步会自动补上。

### 4. 逐仓库同步

对 `repos.txt` 中每个**未注释、非空**的路径，按以下顺序执行：

#### 4.1 前置检查

```
路径为空或以 # 开头 → 跳过
路径下无 .git 目录 → 记录日志 "SKIP not a git repo"，跳过
```

#### 4.2 确定目标分支

1. 在 `branches.txt` 中查找该仓库的配置行
2. 找到 → 使用配置的分支名
3. 未找到 → 使用 `git symbolic-ref --short HEAD` 获取当前分支
4. 无法检测分支 → 记录日志并跳过

#### 4.3 切换分支

```
目标分支 == 当前分支 → 无操作
目标分支 != 当前分支 → 执行 git checkout <branch>
                       失败 → 记录错误，跳过该仓库
```

#### 4.4 暂存变更

```
git add -A
```

将所有变更（新增、修改、删除）加入暂存区。

#### 4.5 提交

```
git diff --cached --quiet → 无变更 → 日志 "Nothing to commit"
有变更 → git commit -m "auto sync <日期> <时间>"
```

提交消息格式示例：`auto sync 2026-05-04 14:30`

#### 4.6 拉取远程更新

```
git pull --rebase --autostash
```

- `--rebase`：将本地提交变基到远程最新提交之上，保持线性历史
- `--autostash`：如果 pull 期间有未提交的脏文件，自动 stash → pull → stash pop

#### 4.7 推送到远程

```
git push
```

- 成功 → 日志 "Pushed"
- 失败 → 日志 "ERROR Push failed"

### 5. 日志处理

每轮同步结束后：

1. **全量日志** (`logs/git-auto-sync.log`)：将本轮日志**前置**写入，保留完整历史
2. **近期日志** (`logs/git-auto-sync-recent.log`)：将本轮日志前置写入，仅保留最近 `KEEP_RECENT` 轮，超出部分截断
3. 清理临时日志文件

### 6. 等待下一轮

```
timeout /t <INTERVAL * 60>    (Windows)
sleep <INTERVAL * 60>         (Linux/macOS)
```

然后回到步骤 1 开始下一轮。

---

## 配置文件说明

| 文件 | 用途 | 是否自动生成 |
|---|---|---|
| `config/repos.txt` | 仓库路径列表，每行一个绝对路径 | 首次运行时创建并打开编辑器 |
| `config/branches.txt` | 仓库-分支映射，控制每个仓库同步哪个分支 | 首次运行时自动生成 |
| `config/sync-settings.txt` | 同步间隔、日志保留轮数 | 需手动创建 |

## 日志文件

| 文件 | 内容 |
|---|---|
| `logs/git-auto-sync.log` | 所有同步轮次的完整日志（按时间倒序） |
| `logs/git-auto-sync-recent.log` | 仅保留最近 N 轮的轻量日志 |

## 单个仓库单轮同步流程图

```
┌─────────────────┐
│  前置检查        │ ─── 无效路径 → 跳过
└────────┬────────┘
         │
┌────────▼────────┐
│  读取分支配置    │ ─── 无配置 → 使用当前分支
└────────┬────────┘
         │
┌────────▼────────┐
│  切换分支        │ ─── 失败 → 跳过
└────────┬────────┘
         │
┌────────▼────────┐
│  git add -A     │
└────────┬────────┘
         │
┌────────▼────────┐
│  git commit     │ ─── 无变更 → 跳过提交
└────────┬────────┘
         │
┌────────▼────────┐
│  git pull       │  (rebase + autostash)
│  --rebase       │
│  --autostash    │
└────────┬────────┘
         │
┌────────▼────────┐
│  git push       │ ─── 失败 → 记录错误
└─────────────────┘
```
